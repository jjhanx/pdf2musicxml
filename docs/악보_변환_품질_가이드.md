# 악보 PDF → MusicXML 품질·호환 가이드 (Audiveris 단일 파이프라인)

업로드한 PDF를 **그대로 Audiveris**에 넘겨 `.mxl` / `.musicxml`을 만듭니다. **별도 OCR·텍스트 선추출·가사 병합 단계는 없습니다.** 가사·글자는 Audiveris가 인식해 MusicXML에 넣을 수 있는 만큼만 포함되며, 인쇄 품질과 동일을 기대하기 어렵습니다.

---

## 서버 배포 후 점검 체크리스트

### 1단계: 코드·빌드

| 순서 | 할 일 | 기대 결과 |
|------|--------|-----------|
| 1-1 | `git pull`(운영 브랜치) | 최신 커밋 |
| 1-2 | `npm ci` 또는 `npm install` | `node_modules` 정합 |
| 1-3 | `npm run build` | `dist/` 생성 |

### 2단계: 환경 변수·프로세스

| 순서 | 할 일 | 기대 결과 |
|------|--------|-----------|
| 2-1 | `AUDIVERIS_BIN`가 실행 파일을 가리키는지 | `GET /api/health`에서 확인 |
| 2-2 | (선택) `AUDIVERIS_NO_FLAT_OUTPUT` — 공통 출력 폴더 레이아웃 조정 시 |
| 2-3 | 변경 후 **Node 재시작** (`pm2 restart` 등) | 환경 반영 |

### 3단계: 헬스·업로드

| 순서 | 할 일 | 기대 결과 |
|------|--------|-----------|
| 3-1 | `curl -sS http://127.0.0.1:<PORT>/api/health` | 200 JSON |
| 3-2 | 작은 PDF로 변환 | 202 → 폴링 → 다운로드 |
| 3-3 | `POST /api/convert` 응답 헤더 `X-Pdf2Mxl-Async: 202-after-upload` | 배포 버전 확인용 |
| 3-4 | nginx 시 `client_max_body_size`, 타임아웃 | README「문제 해결」참고 |

### 4단계: 한글 파일명·ZIP

| 순서 | 할 일 | 기대 결과 |
|------|--------|-----------|
| 4-1 | 한글 파일명 PDF + **디버그 ZIP** | ZIP 안 업로드 원본 PDF 이름이 가능한 한 보존 |
| 4-2 | 이름 깨짐 지속 | 브라우저·역프록시의 `Content-Disposition` 처리 확인 |

### 5단계: 디버그 ZIP 내용

디버그 모드를 켜면 예상되는 내용은 다음과 같습니다.

- **업로드한 원본 PDF** (파일명은 업로드 시 이름과 최대한 동일)
- Audiveris가 만든 **`.mxl` / `.musicxml`** 파일들

---

## Audiveris가 MXL을 만들지 못할 때 (HTTP 422, WARN·ERS)

출력 폴더에 `.mxl`/`.musicxml`이 **하나도 없으면** 작업이 실패합니다.

- 로그에 `WARN [파일이름#10]` 형태가 보이면 `#10`은 보통 **10번째 악보 장(sheet)** 문제를 가리킵니다. 한 장이라도 내보내기가 막히면 결과가 비어 보일 수 있습니다.
- **ERS** 등은 Audiveris 내부 단계 관련 메시지가 로그에 붙는 경우가 많습니다.

**직접 확인할 것**

1. 동일 PDF를 **Audiveris GUI**로 열어 문제 장을 수동으로 점검합니다.
2. `pm2 logs` 등으로 **전체 stdout/stderr**에서 `WARN`, `ERROR`, `ERS`와 장 번호를 확인합니다.
3. PDF가 **저해상도 스캔**이거나 가사·글자가 오선과 겹치면 인식이 불안정할 수 있습니다 — 원본 품질·출처를 검토합니다.

---

## `realValue` / mxlplayer 오류

Audiveris **MusicXML 4.x** 출력은 일부 웹 뷰어가 완전히 지원하지 않을 수 있습니다.

- **대응**: [MuseScore](https://musescore.org/) 등에서 열고 **MusicXML 3.1** 등으로 다시 내보낸 뒤 mxlplayer에서 시험합니다.

---

## 마디 수·파트가 줄어 보일 때

인식 실패, 파트 매핑, 일부 `.mxl`만 성공하는 경우가 있습니다. 디버그 ZIP의 **여러 부품 파일**(예: `mvt1`, `mvt2`)을 각각 열어 보고, Audiveris 로그와 원본 PDF를 대조합니다.
